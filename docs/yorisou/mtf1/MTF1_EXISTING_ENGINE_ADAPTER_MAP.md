## Adapter build order (implementation-package guidance, not executed here)

1. `rule_based` adapter (C02/F01/F02) — closest fit, three methods at once.
2. RF + LD + micro adapters — same family of tag scoring, adds persistence parity.
3. `imairo-120q` adapter — byte-compatible wrap of the protected pipeline.
4. New-format capabilities (repeated_check for daily-check-in; two_person for relationship-pair-check; image/color choice) — these are new engine work, not adaptation.
5. S01 classification adapter.
