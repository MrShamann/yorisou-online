## Purpose

Collects the Japanese user-facing copy required for the volunteer pilot, in the order a candidate would encounter it: short invitation → full explanation → participation conditions → pilot explanation → manual review pending → delivery message → feedback request → hold message → non-use/deletion acknowledgment.

Tone: simple, calm, unhurried, no pressure, no urgency. Declining is always presented as a normal, equal-weight choice.

---

## 1. Short Invitation

**Context:** Initial outreach from Edward to a hand-selected candidate. Sent in a conversation channel, not a public form.

```
Yorisouで「詳細レポート」というものを準備しています。
今回は、その読みやすさと安全性を確認するための、少人数のテスト読者を募集しています。

支払いは発生しません。
協力いただけるかどうかは、自由にお決めください。お断りいただいてもかまいません。
詳しい内容と進め方を、ご希望があればお伝えします。
```

---

## 2. Full Explanation

**Context:** Shown after the candidate expresses interest. Explains what the pilot is and what it is not.

```
Yorisou詳細レポートは、Yorisouのチェック結果をもとに、
いまの状態を、感情・関係・行動・回復の視点から整理する自己理解レポートです。

このテストでは、

- 今回のYorisouチェックの回答をもとにレポートを作ります
- 作成したレポートは、担当者(Edward)が一つひとつ読み確認したうえでお渡しします
- お渡し後、読みやすさや安全性について、感想をいくつかお伺いします

このレポートは、医療・心理診断、治療、カウンセリング、未来予測ではありません。
将来の出来事を当てるものでもありません。
今の状態を整理し、小さな次の行動を考えるための読み物です。

支払いは発生しません。
将来、有料でご提供する場合の価格や条件は、現時点では決まっていません。
今回のご協力を、将来の購入の前提として扱うこともありません。
```

---

## 3. Participation Conditions

**Context:** Shown together with the full explanation. Sets expectations and exclusions plainly.

```
ご参加にあたって、いくつか前提をお伝えします。

- 今回のYorisouチェックの回答と、結果から見えるあなたの傾向のみを使います
- 過去のYorisouチェック結果、LINEでのやりとり、長期メモリーは使いません
- ご本人のお名前や、ご連絡先などの個人情報は、レポート作成に使いません
- レポートの内容を、AIモデルの学習に使うこともありません
- レポートは、担当者の確認を経てお渡しするため、当日中のお渡しは難しい場合があります
- ご感想は、お答えにくいものはとばしていただいてかまいません
- いつでも参加を取り下げることができます

以下に当てはまる方は、今回のテストとは別の道のほうが合うことがあります。

- いま、強いつらさや危機的な状況にある方
- 安全でない、または相手のふるまいが怖いと感じる関係にある方
- いま治療や専門的な支援を受けていて、今回のレポートを取り入れることで支援に影響が出る可能性のある方

ご自身で迷われる場合は、参加を見合わせていただいてかまいません。
```

## 4. Pilot Explanation (Consent Step Lead-In)

**Context:** Just before the consent step. A short bridge that sets the next step's purpose.

```
このあと、利用範囲とレポートの位置づけについて、二つのご確認をお願いします。
それぞれ、内容を読んでからご同意いただける場合のみ進みます。
ご同意いただかない選択もできます。その場合は、レポート作成には進みません。
```

---

## 5. Manual Review Pending Message

**Context:** Shown after the candidate requests the report.

```
レポート作成のご依頼を受け付けました。
担当者がひとつずつ確認してお届けするため、お時間をいただきます。
ご準備ができ次第、ご連絡します。
お届けまでの時間が長くなりそうな場合は、その旨もご連絡します。
```

---

## 6. Delivery Message

**Context:** Sent with the delivered report.

```
詳細レポートをお届けします。
お時間のあるときに、ゆっくり読んでみてください。

このレポートは、いまの状態を整理するためのものです。
医療・心理診断、治療、カウンセリング、未来予測ではありません。
読んでいて気になる点や、合わないと感じる部分は、無理に当てはめなくて大丈夫です。

読み終えたあとに、いくつかご感想をうかがいます。
お答えにくいものはとばしてかまいません。
ご感想をお寄せいただかない選択もできます。
```

---

## 7. Feedback Request

**Context:** Shown after delivery. Explicit that feedback is optional, internal use only.

```
お読みいただきありがとうございました。
いくつかご感想をお聞かせいただけたらうれしいです。

ご回答は任意です。お答えにくいものはとばしてかまいません。
ご感想は、内部のレポート品質改善のためにのみ使います。
お名前を出した形での公開や、宣伝・お客様の声としての利用は行いません。

自由記述欄には、お名前、相手の方の特定につながる情報、健康・危機の詳細、
LINE やメッセージのやり取り内容を含めないようお願いします。
```

---

## 8. Hold Message

**Context:** Used when a report cannot be delivered yet due to internal review or safety confirmation. Neutral copy. Never blames the user.

```
安全確認と内容確認のため、このレポートは現在お渡しを保留しています。
必要な確認が終わり次第、案内します。
お時間をいただき申し訳ありません。
```

If the hold cannot be resolved within a reasonable window:

```
今回はお約束したレポートをお届けすることが難しくなりました。
ご協力いただいたお時間に、あらためてお礼申し上げます。

このテストは、レポートの品質と安全性を確認するためのものです。
品質や安全面の確認に時間が必要な場合は、お渡しを見合わせることがあります。
ご提供いただいた情報の今後の利用を希望されない場合は、止めることができます。
削除のご依頼も承ります。ご希望があればお知らせください。
```

## 9. Non-Use / Deletion Request Acknowledgment

**Context:** Sent in reply to a non-use or deletion request.

```
ご依頼を受け付けました。

「今後の利用を止める」をご希望の場合は、
これ以降、レポート作成や見直しの目的で使うことを止めます。

「削除を依頼する」をご希望の場合は、
対応できる範囲を確認のうえご連絡します。
すべての情報がただちに、または完全に削除されるわけではない場合があります。
管理上必要な記録(操作ログなど)は、対応の範囲外で残ることがあります。

確認が必要な場合は、こちらからご連絡します。
```

## 10. Load-Bearing Wording Rules

These wording rules apply across all blocks above. The pilot's user-facing copy must reflect each rule.

| Rule | Required wording / posture |
|---|---|
| **No payment** | "支払いは発生しません" stated explicitly at invitation, explanation, and on request acknowledgment |
| **No emergency support** | Pilot copy does not present Yorisou as a crisis or emergency resource; users in crisis are directed (in the participation conditions) to other support |
| **No sensitive free-text details** | Free-comment warning lists the categories not to include; submissions are redacted before storage |
| **Manual review delay** | "お時間をいただきます" / "当日中のお渡しは難しい場合があります" — honest about manual review timing; no automation language |
| **User can stop / request non-use** | Stated at full explanation and at non-use acknowledgment |
| **No overpromise on deletion** | "対応できる範囲" / "ただちに、または完全に削除されるわけではない場合があります" — never "すぐに削除" or "完全に消去" |
| **No delivery time guarantee** | The pilot does not promise a delivery window unless one is later separately approved |
| **Not diagnosis / therapy / counseling / fortune-telling / future prediction / professional advice** | Stated in full explanation and delivery message; reinforced by Section 13 safety note in the report itself |
