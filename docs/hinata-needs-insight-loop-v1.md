# Hinata Elder-Needs Insight Loop v1

- Needs signals: use [data/hinata-needs-signal-schema-v1.json](/Users/yangjin/yorisou-online/data/hinata-needs-signal-schema-v1.json) and [lib/server/openclawNeedsInsight.ts](/Users/yangjin/yorisou-online/lib/server/openclawNeedsInsight.ts) to extract cautious user-need signals from conversations without jumping straight to product intent.
- Runtime grounding: [lib/server/hinataKnowledge.ts](/Users/yangjin/yorisou-online/lib/server/hinataKnowledge.ts) now pulls compact need-lens, playbook, option-hint, and example snippets from the domain pack into the existing knowledge packet path for `/api/support/chat`.
- Aggregation: use [data/hinata-insight-aggregation-schema-v1.json](/Users/yangjin/yorisou-online/data/hinata-insight-aggregation-schema-v1.json) with `buildNeedsInsightAggregate(...)` for lightweight recurring-pattern review.
- Feedback mapping: use [data/hinata-feedback-mapping-v1.json](/Users/yangjin/yorisou-online/data/hinata-feedback-mapping-v1.json) to connect recurring need patterns to possible product, service, design, and content implications.
- Domain-pack alignment: treat [data/hinata-domain-needs-taxonomy-v1.json](/Users/yangjin/yorisou-online/data/hinata-domain-needs-taxonomy-v1.json) as the upstream need layer, [data/hinata-product-service-catalog-v1.json](/Users/yangjin/yorisou-online/data/hinata-product-service-catalog-v1.json) as a downstream solution layer, and [data/hinata-matching-playbook-v1.json](/Users/yangjin/yorisou-online/data/hinata-matching-playbook-v1.json) as the timing bridge between them.
- Logging: [lib/server/openclawReflection.ts](/Users/yangjin/yorisou-online/lib/server/openclawReflection.ts) now keeps reflection artifacts separate from cautious needs-signal artifacts, which are stored through [lib/server/openclawNeedsInsight.ts](/Users/yangjin/yorisou-online/lib/server/openclawNeedsInsight.ts) in `phase1-openclaw-needs-signals.json`.
- Artifact paths: runtime writes still land in local/runtime files under `/tmp/yorisou-phase1` in production-like mode, while [lib/server/openclawArtifactStore.ts](/Users/yangjin/yorisou-online/lib/server/openclawArtifactStore.ts) can mirror per-artifact JSON into the shared store at `phase1/openclaw-artifacts/...` when `YORISOU_SHARED_STORE_BUCKET` is configured.
- Protected inspection: [app/api/admin/openclaw-artifacts/route.ts](/Users/yangjin/yorisou-online/app/api/admin/openclaw-artifacts/route.ts) provides a protected recent-artifact read path for reflection and needs-signal artifacts. It accepts admin session access or `OPENCLAW_ARTIFACT_READ_TOKEN`, and supports `type`, `limit`, `since`, and `id`.
- Review flow: run `npm run review:needs` to read emitted needs-signal artifacts from local/runtime files plus mirrored-compatible sources, build an aggregate summary, derive feedback candidates, and write a machine-readable report to `test-results/hinata-needs-review.json`. Keep business conclusions human-reviewed.
