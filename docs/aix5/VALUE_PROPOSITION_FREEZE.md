# AIX-5 — Value Proposition Freeze (binding)

Produced with `ai-native-product-architect` + `premium-ui-ux-creative-director` +
`product-marketing` review. This is the **binding** product proposition for
AIX-5 and every downstream surface. It supersedes the AIX-4 positioning, which
the Founder rejected for merge because the value proposition was not clearly
defined across the product (YORISOU could still be read as a sophisticated test
site).

## The one product truth

**YORISOU understands the person in order to provide continuing, personalized
support beyond the initial check.** YORISOU does **not** assign a fixed type. It
continually understands the user's current state, changes and preferences, and
uses that understanding to help the user find more suitable information, content,
experiences, services, relationships/communities, public resources, products and
practical next steps. The check is only one input; the product value is the
continuing personalized support that follows.

## Frozen Japanese messaging hierarchy

- **Main proposition:** 今のあなたを知り、これからを一緒に選ぶ。
- **Product definition:** YORISOUは、あなたの状態・変化・好みを理解し、必要な情報、体験、サービス、つながりを、必要なときに届けるAI-native伴走プラットフォームです。
- **Role of the check:** チェックは、あなたを理解するための最初の入口のひとつです。

## Target users

People (Japan-first, consumer) who feel their current state — mood, rhythm,
relationships, work-life, recovery — is hard to put into words, and who want a
calmer way to notice it and then find what actually fits, at their own pace,
without being labelled or sold to.

## Primary problem

Existing tools either stop at a label/result (personality tests) or push a
catalogue at the user. Nobody continuously understands the person and then helps
them find the *next fitting thing* — information, a report, an experience, a
service, a connection — as their state changes.

## Product promise (four things YORISOU does)

- **A. Understands me** — from explicit, permissioned inputs: checks, choices,
  saved interests, feedback, stated needs, changes over time.
- **B. Remembers what I choose to preserve** — user-controlled continuity: saved
  results/intentions, return paths, private history, LINE continuity where chosen.
- **C. Helps me find what fits** — information, content, reports, actions,
  experiences, places, public resources, services, qualified products, optional
  human connection — each explained and optional.
- **D. Learns from outcomes with permission** — signals: useful / tried / save /
  revisit / not-for-me / feedback / optional prototype participation.

## Why understand-first, why continue

Understanding is the prerequisite for relevance: without a current-state read,
any recommendation is generic. And because state changes, a one-time read is
insufficient — the value compounds only if YORISOU remembers, recommends, learns
and welcomes the user back. The **loop is the product**, not the test.

## Differentiators

- Continuous, permissioned **understanding** → relevance, not a fixed type.
- **Continuity by design** (remember + return), not a one-shot result.
- Recommendations are **explained and optional** — never paid placement, never a
  marketplace, never pressure.
- Calm, non-clinical, non-deterministic; the user keeps the initiative.

## Prohibited interpretations (primary public copy must NOT imply)

A personality test · a diagnosis · a fixed type result · a report product · a
recommendation marketplace / paid placement · a LINE bot · a community website ·
that a result defines the user · that YORISOU autonomously knows private facts ·
that unfinished capability is already operational.

## CTA hierarchy (frozen)

1. **Primary — begin from the user's current state:** 今の自分から始める (→ /check-in).
2. **Secondary — explore what YORISOU provides:** YORISOUでできることを見る (→ /#system).
3. **Continuity:** 保存 / 戻る / LINEで続ける / 見返す.
4. **Discovery:** おすすめ / レポート / 体験 / 資源を見る.
5. **Participation:** 感想を送る / 共創に参加 / パートナー相談.

The first check is free, no-login, and **one** starting method — mentioned only
after the platform proposition, never at hero-headline weight, and "120問" is
never given hero-level visual weight.
