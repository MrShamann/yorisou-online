## 1. Purpose

This register tracks governed execution from approved decision to verified production state.

## 2. Required Fields

| Field | Required content |
|---|---|
| Task ID | Stable identifier |
| Parent decision | Decision ID or policy basis |
| Capability | User or operational capability |
| Current status | Current / Prototype / Planned / Deferred |
| Executor | Codex or approved human |
| Approver | Edward |
| Repository scope | Repo, branch, and intended files |
| Acceptance criteria | Observable outcomes |
| Safety/privacy checks | Required checks |
| Validation | Commands, tests, screenshots, or production check |
| PR/status | Link or state; no automatic merge authority |
| Result | Passed / Failed / Blocked / Superseded |
| Follow-up | Remaining work |

## 3. Execution Rules

- Codex is the formal repository execution agent.
- The founder performs final approval and merge decisions.
- Tasks must not claim completion without evidence.
- Production database changes require explicit migration and verification steps.
- UI completion requires mobile and Japanese-language review.
- Governance-sensitive changes must cite the controlling policy.
