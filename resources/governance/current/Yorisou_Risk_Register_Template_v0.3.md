## 1. Purpose

This register records material product, user, legal, privacy, content, commercial, technical, and execution risks.

## 2. Required Fields

| Field | Required content |
|---|---|
| Risk ID | Stable identifier |
| Date opened | YYYY-MM-DD |
| Owner | Responsible person or agent |
| Area | Product / Privacy / Content / AI / Security / Commercial / Legal / Operations |
| Description | Concrete risk event |
| Trigger | Observable condition |
| Likelihood | Low / Medium / High |
| Impact | Low / Medium / High / Critical |
| Current controls | Existing prevention or response |
| Required action | Specific next action |
| Due date | Date or review milestone |
| Status | Open / Mitigating / Accepted / Closed |
| Evidence | Links, tests, logs, or decision record |

## 3. Mandatory v0.3 Risk Categories

The live register must include at least:

- privacy leakage through experience cards;
- spam, scams, harassment, and impersonation;
- unsafe recommendations;
- AI fabrication or material rewriting;
- underage-user privacy and interaction risk;
- LINE/email account-linking error;
- deletion and visibility failure;
- misleading commercial disclosure;
- low-quality cold-start content;
- moderation workload for a one-person company;
- data breach or credential exposure;
- capability overstatement;
- migration and deployment failure.

## 4. Acceptance Rule

Critical risks cannot be silently accepted. Acceptance requires founder decision, rationale, monitoring condition, and review date.
