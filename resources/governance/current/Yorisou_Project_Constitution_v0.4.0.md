# YORISOU Project Constitution v0.4.0

**Status:** Approved
**Pack:** YORISOU Governance Pack v0.4.0 · **Drafted:** 2026-07-13 · **Approved as activation candidate:** 2026-07-14 · **Approver:** Edward

## 1. Purpose
Define what YORISOU is, who governs it, and the invariants no other document may override.

## 2. Scope
All YORISOU activity: product, platform, agents, commerce, research, external representation.

## 3. Binding rules
1. YORISOU is one governed core platform (brand yorisou.online), Japan-first, Japanese-first, delivered through Web and LINE/LIFF channel adapters; iOS/Android remain planned and must not be described as available.
2. The core product identity is the **Calm Emotional Companionship System** built on exactly **six Core Systems**: Emotional Value; Relationship & Companion; Personal Archive & Memory; Recommendation & Reflection; Physical–Digital Expression; Life Continuity & Legacy.
3. The **Consent-Based Personal Context Layer is a cross-cutting constraint on all six systems and is not a seventh Core System**. The **Founder Dashboard is an operations/evidence surface, not a Core System, parent engine, or user-facing product center.**
4. Edward is the sole Founder authority (see Founder Authority and Decision Rights v1.0). Governance documents bind all agents and all implementation.
5. User dignity invariants: memory only by explicit consent; use only by separate permission; deletion honored absolutely; no engagement-pressure mechanics in memory or companion loops; the product must remain safe to leave.
6. Truth invariant: synthetic or internal evidence is never represented as human or market validation.

## 4. Prohibited behavior
Adding Core Systems; demoting the Consent Layer to a feature; making the Founder Dashboard user-facing; any dark-pattern retention mechanics; claiming validation states not evidenced.

## 5. Decision authority
Amendment of this constitution: Edward only, with a written decision record.

## 6. Required evidence
Every constitutional claim about implementation status must trace to the Implementation Baseline document with evidence anchors.

## 7. Exceptions
None. Constitutional invariants have no exception path; anything requiring one is an amendment.

## 8. Change control
Changes only via Change Management v1.0 (decision record + pack re-checksum + archive of prior text).

## 9. Relationship to predecessor documents
Succeeds Yorisou_Project_Constitution_v0.3.1 (RETAIN_WITH_AMENDMENT). Retained: one-platform doctrine, channel-adapter model, Japan/Japanese-first, Founder authority. Amended: product identity clauses now name Calm Emotional Companionship and the six Core Systems. Superseded: test-product-centric identity wording.
