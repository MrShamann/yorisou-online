# YORISOU Experience Architecture Doctrine v1.0

**Status:** Approved
**Pack:** YORISOU Governance Pack v0.4.0 · **Drafted:** 2026-07-13 · **Approved as activation candidate:** 2026-07-14 · **Approver:** Edward

## 1. Purpose
Bind the internally validated experience (Phase 1.2 accepted build) as the normative experience law.

## 2. Scope
All user-facing experience design, copy, and interaction in companion surfaces; tone rules extend to acquisition surfaces.

## 3. Binding rules
1. The validated journey is normative: entry value proposition → optional Quiet Presence (optional mood entry, skippable) → capture → meaning → MemoryCandidate review → confirm → separate UsePermission → safe exit → voluntary 7–14 day return → reflection with the four choices (今も大切/少し変わった/もう手放してよい/まだ分からない) → controls → Trust Repair when needed.
2. Japanese-first, calm, low-pressure copy per the accepted copy set; AI disclosure on first surface; why-appeared transparency wherever a memory is referenced.
3. No silent no-ops: every blocked action explains itself (D-003 rule).
4. Memory-state visibility: users can always see the state of their memories and relationship condition.
5. Accessibility and cognitive-load standards of the Experience Architecture V1 sources apply.

## 4. Prohibited behavior
Therapy/diagnostic language; dependency cues; guilt for absence; engagement pressure; surveillance language; technical jargon in user copy; removing safe exit; forced mood disclosure.

## 5. Decision authority
Experience-law changes: Edward with experience-review evidence. Copy-level edits: change-managed with Japanese language review.

## 6. Required evidence
Browser-level evidence (screenshots + DOM assertions) for every experience change, per the established evidence discipline.

## 7. Exceptions
Bounded A/B copy variation is allowed within tone rules, never on consent-bearing copy.

## 8. Change control
Change Management v1.0.

## 9. Relationship to predecessor documents
Succeeds the UX-boundary portions of Product Doctrine v0.3.1 (SUPERSEDE). Source lineage: Experience Architecture V1, MVP Flow Spec V1, validated Phase-1.2 build (Founder-accepted).
